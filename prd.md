# 社交练习生 App MVP PRD

## 1. 产品定位

### 1.1 产品一句话定义

一个面向 20-30 岁年轻人的安卓文字对话陪练 App，通过模拟中国现代真实社交场景，让用户反复练习开口、接话、分寸感、拒绝和收尾，并在对话后获得可执行反馈。

### 1.2 MVP 目标

MVP 不做完整社交教育平台，只验证一个核心闭环：

1. 用户选择一个真实场景。
2. App 生成明确的对方角色和对话目标。
3. 用户与 AI 进行 6-10 轮文字对话。
4. App 根据整段对话输出复盘。
5. 用户收藏可复用表达。

### 1.3 产品边界

MVP 是自用工具，不考虑商业化、增长、社区和复杂账号体系。第一版优先保证：

- 场景真实。
- 对话能推进。
- 复盘具体，不空泛。
- 历史记录可回看。
- 好表达可沉淀。

### 1.4 推荐实现形态

- 平台：Android。
- 交互：文字输入为主。
- AI 能力：通过大语言模型 API 完成角色扮演、对话推进和复盘。
- 数据：本地存储为主。
- 账号：MVP 不做登录。

## 2. 目标用户

### 2.1 主要用户

20-30 岁年轻人，不限制职业。用户可能是学生、职场新人、自由职业者、刚到新城市生活的人，或日常社交经验不足的人。

### 2.2 用户特征

- 面对陌生人或半熟人时不知道怎么开口。
- 担心自己说话尴尬、没分寸、显得冷淡或讨好。
- 想提升社交能力，但不愿意找真人练习。
- 希望通过重复练习获得更自然的表达方式。
- 对中国式饭局、亲戚寒暄、朋友局、同事局等场景有现实压力。

### 2.3 用户核心问题

| 问题 | 具体表现 | App 需要训练的能力 |
| --- | --- | --- |
| 不敢开口 | 看到陌生人不知道第一句话说什么 | 开场 |
| 不会接话 | 只会回复“嗯”“还好”“哈哈” | 延展话题 |
| 分寸不稳 | 过度暴露、过度讨好、太生硬 | 分寸感 |
| 不会拒绝 | 被劝酒、被问隐私、被要求帮忙时僵住 | 边界表达 |
| 不会收尾 | 想结束对话但不知道怎么说 | 自然结束 |
| 缺少复盘 | 不知道自己具体哪里说得不好 | 反馈理解 |

## 3. 核心场景

### 3.1 场景设计原则

每个场景必须包含：

- 场景背景：用户身处什么情境。
- 对方角色：AI 扮演谁。
- 社交压力：这个场景为什么难。
- 训练目标：本轮主要练什么。
- 对话触发点：AI 在过程中可能抛出的挑战。
- 结束条件：什么时候可以复盘。

### 3.2 MVP 场景列表

| 场景 ID | 分类 | 场景名称 | AI 角色 | 训练目标 | 典型压力点 |
| --- | --- | --- | --- | --- | --- |
| S001 | 陌生人 | 朋友聚餐里有不熟的人 | 朋友的朋友 | 开场、找共同话题 | 对方回答简短 |
| S002 | 陌生人 | 和朋友的朋友第一次见面 | 外向但不熟的人 | 自我介绍、接话 | 对方开玩笑试探 |
| S003 | 陌生人 | 活动中坐在陌生人旁边 | 同桌陌生人 | 低压力开场 | 不知道聊什么 |
| S004 | 半熟人 | 新加入一个已经熟悉的小圈子 | 小圈子成员 | 融入、观察话题 | 大家有内部梗 |
| S005 | 半熟人 | 和不太熟的同学或同事单独聊天 | 半熟同龄人 | 延展话题 | 一问一答冷场 |
| S006 | 半熟人 | 老同学多年后见面 | 老同学 | 寒暄、近况交换 | 攀比、尴尬沉默 |
| S007 | 职场 | 第一次参加公司聚会 | 公司同事 | 分寸感、融入 | 不知道聊工作还是生活 |
| S008 | 职场 | 和领导或前辈同桌吃饭 | 领导/前辈 | 尊重但不讨好 | 被问工作适应情况 |
| S009 | 职场 | 被同事开玩笑或试探 | 老员工同事 | 稳定回应 | 对方半开玩笑半试探 |
| S010 | 亲戚 | 走亲戚被问对象 | 亲戚长辈 | 委婉回应、边界感 | 追问恋爱婚姻 |
| S011 | 亲戚 | 被问工资、买房、未来规划 | 亲戚长辈 | 隐私边界、转移话题 | 对方持续追问 |
| S012 | 亲戚/饭局 | 被劝酒或被强行安排帮忙 | 长辈/熟人 | 体面拒绝 | 对方施压“给个面子” |

### 3.3 场景验收要求

每个场景必须能单独配置以下内容：

- `scenario_id`
- `title`
- `category`
- `background`
- `user_goal`
- `ai_role_name`
- `ai_role_profile`
- `difficulty`
- `opening_message`
- `challenge_points`
- `evaluation_focus`
- `suggested_rounds`

## 4. MVP 功能列表

### 4.1 功能总览

| 模块 | 功能 | MVP 是否实现 | 说明 |
| --- | --- | --- | --- |
| 场景库 | 固定 12 个场景 | 是 | 不做用户自定义复杂场景 |
| 场景详情 | 查看背景、角色、目标 | 是 | 开始练习前展示 |
| 对话练习 | 文字对话 | 是 | 用户输入，AI 回复 |
| 对话控制 | 结束练习 | 是 | 用户可主动结束 |
| 自动复盘 | 根据对话生成反馈 | 是 | 对话结束后触发 |
| 历史记录 | 查看过往练习 | 是 | 本地存储 |
| 表达收藏 | 收藏推荐表达 | 是 | 从复盘页收藏 |
| 设置 | API 配置 | 是 | 自用版本需要配置模型服务 |
| 新手引导 | 简单说明 | 可选 | 首次进入展示一次 |

### 4.2 开发任务拆分

#### 4.2.1 场景库

- 创建本地场景配置文件或内置数据表。
- 实现场景列表读取。
- 支持按分类筛选。
- 支持进入场景详情页。

#### 4.2.2 对话练习

- 创建练习会话。
- 根据场景生成系统提示词。
- 展示 AI 开场白。
- 支持用户输入文本。
- 调用 AI 获取下一轮回复。
- 保存用户消息和 AI 消息。
- 支持手动结束练习。
- 到达建议轮数后提示用户可以复盘。

#### 4.2.3 复盘

- 将场景信息和完整对话传给 AI。
- 生成结构化复盘结果。
- 展示评分、问题、改进建议和替代表达。
- 保存复盘结果。
- 支持从替代表达中收藏句子。

#### 4.2.4 历史记录

- 按时间倒序展示练习记录。
- 展示场景名称、练习时间、评分摘要。
- 支持进入历史复盘详情。
- 支持删除单条记录。

#### 4.2.5 表达库

- 展示已收藏表达。
- 支持按标签筛选。
- 支持复制表达。
- 支持删除收藏。

#### 4.2.6 设置

- 配置 AI API 地址。
- 配置 API Key。
- 配置模型名称。
- 测试连接。
- 清空本地数据。

## 5. 页面结构

### 5.1 页面层级

```text
App
├── 练习 Tab
│   ├── 场景列表页
│   ├── 场景详情页
│   ├── 对话练习页
│   └── 复盘结果页
├── 复盘 Tab
│   ├── 历史记录页
│   └── 历史复盘详情页
├── 表达库 Tab
│   └── 表达列表页
└── 设置页
    ├── API 配置
    └── 数据管理
```

### 5.2 底部导航

MVP 底部导航保留 3 个主入口：

| Tab | 页面 | 目的 |
| --- | --- | --- |
| 练习 | 场景列表页 | 开始新练习 |
| 复盘 | 历史记录页 | 回看问题和建议 |
| 表达库 | 表达列表页 | 沉淀可复用表达 |

设置入口放在页面右上角，不放底部 Tab。

## 6. 每个页面的字段、按钮、交互

### 6.1 场景列表页

#### 字段

- 页面标题：`练习`
- 分类筛选：`全部`、`陌生人`、`半熟人`、`职场`、`亲戚/饭局`
- 场景卡片：
  - 场景名称
  - 场景分类
  - 一句话背景
  - 训练目标标签
  - 难度

#### 按钮

- `场景卡片`：点击进入场景详情。
- `设置图标`：进入设置页。

#### 交互

- 默认展示全部场景。
- 点击分类后，只展示对应分类场景。
- 点击场景卡片进入场景详情页。
- 如果 API 未配置，点击场景卡片仍可进入详情，但开始练习时提示配置。

### 6.2 场景详情页

#### 字段

- 场景名称
- 场景背景
- AI 扮演角色
- 本轮训练目标
- 可能遇到的问题
- 建议轮数
- 难度

#### 按钮

- `开始练习`
- `返回`

#### 交互

- 点击 `开始练习`：
  - 若 API 配置有效，创建新会话并进入对话练习页。
  - 若 API 未配置，弹出提示，引导进入设置页。

### 6.3 对话练习页

#### 字段

- 顶部信息：
  - 场景名称
  - AI 角色名称
  - 当前轮数
- 消息区：
  - AI 消息气泡
  - 用户消息气泡
  - 加载状态
  - 错误状态
- 输入区：
  - 文本输入框
  - 发送按钮
- 底部辅助：
  - `结束并复盘`

#### 按钮

- `发送`
- `结束并复盘`
- `返回`

#### 交互

- 进入页面后展示 AI 开场白。
- 用户输入为空时，`发送` 按钮不可用。
- 用户点击 `发送`：
  - 保存用户消息。
  - 禁用输入框。
  - 展示 AI 回复加载状态。
  - 成功后展示 AI 回复并恢复输入。
  - 失败后展示重试按钮。
- 当前轮数达到建议轮数后，提示可以结束复盘，但不强制结束。
- 点击 `结束并复盘`：
  - 若用户消息少于 3 条，提示“对话太短，复盘可能不准确”，用户可继续或仍然复盘。
  - 触发复盘生成，进入复盘结果页。

### 6.4 复盘结果页

#### 字段

- 场景名称
- 练习时间
- 总体评价
- 维度评分：
  - 敢开口
  - 接话能力
  - 分寸感
  - 话题推进
  - 自然度
- 暴露的问题列表
- 关键片段分析：
  - 用户原话
  - 问题说明
  - 更好说法
- 可收藏表达：
  - 表达内容
  - 使用场景标签
- 下次练习建议

#### 按钮

- `收藏表达`
- `再练一次`
- `返回场景列表`
- `查看完整对话`

#### 交互

- 复盘生成中展示加载状态。
- 复盘成功后保存到本地历史。
- 点击 `收藏表达` 后写入表达库，并将按钮状态改为 `已收藏`。
- 点击 `再练一次` 创建同场景新会话。
- 点击 `查看完整对话` 展开本次对话记录。

### 6.5 历史记录页

#### 字段

- 页面标题：`复盘`
- 历史记录卡片：
  - 场景名称
  - 练习时间
  - 总体评分
  - 主要问题摘要
  - 对话轮数

#### 按钮

- `历史记录卡片`
- `删除`

#### 交互

- 默认按练习时间倒序。
- 点击记录进入历史复盘详情页。
- 删除前二次确认。
- 无记录时展示空状态。

### 6.6 历史复盘详情页

#### 字段

- 与复盘结果页一致。
- 额外展示：
  - 完整对话记录
  - 已收藏表达状态

#### 按钮

- `收藏表达`
- `删除记录`
- `再练一次`

#### 交互

- 点击 `再练一次` 使用同一场景重新创建会话。
- 删除记录后返回历史记录页。

### 6.7 表达列表页

#### 字段

- 页面标题：`表达库`
- 标签筛选：
  - `全部`
  - `开场`
  - `接话`
  - `拒绝`
  - `转移话题`
  - `收尾`
  - `长辈`
  - `职场`
- 表达卡片：
  - 表达内容
  - 适用场景
  - 来源场景
  - 收藏时间

#### 按钮

- `复制`
- `删除`

#### 交互

- 点击标签筛选表达。
- 点击复制后复制到剪贴板并提示成功。
- 删除前二次确认。
- 无收藏时展示空状态。

### 6.8 设置页

#### 字段

- API 地址
- API Key
- 模型名称
- 连接状态
- 本地数据占用

#### 按钮

- `保存配置`
- `测试连接`
- `清空练习记录`
- `清空表达库`
- `返回`

#### 交互

- API Key 输入框默认隐藏明文，可切换显示。
- 点击 `测试连接` 后调用一次轻量请求。
- 测试成功显示成功状态。
- 测试失败显示错误原因。
- 清空数据前必须二次确认。

## 8. 异常状态

### 8.1 API 未配置

- 触发位置：开始练习、发送消息、生成复盘。
- 页面表现：弹窗提示“需要先配置 AI 服务”。
- 用户操作：
  - `去设置`
  - `取消`

### 8.2 API 请求失败

- 触发位置：AI 回复、生成复盘、测试连接。
- 可能原因：
  - 网络不可用
  - API Key 错误
  - 服务超时
  - 模型返回格式错误
- 页面表现：
  - 对话页展示错误提示和 `重试`。
  - 复盘页展示失败状态和 `重新生成`。

### 8.3 AI 返回内容不可用

- 触发条件：
  - 返回空内容。
  - 返回与场景无关内容。
  - 复盘 JSON 解析失败。
- 处理方式：
  - 对话回复失败时允许重试。
  - 复盘解析失败时重新请求一次。
  - 第二次仍失败，展示原始文本摘要，并标记本次复盘异常。

### 8.4 对话太短

- 触发条件：用户消息少于 3 条时点击 `结束并复盘`。
- 页面表现：弹窗提示“当前对话较短，复盘可能不准确”。
- 用户操作：
  - `继续对话`
  - `仍然复盘`

### 8.5 本地数据为空

- 历史记录为空：
  - 展示“还没有练习记录”。
  - 按钮：`去练习`
- 表达库为空：
  - 展示“还没有收藏表达”。
  - 按钮：`去练习`

### 8.6 本地存储失败

- 触发位置：保存消息、保存复盘、收藏表达。
- 页面表现：Toast 或 Snackbar 提示。
- 处理方式：
  - 消息保存失败时不中断当前页面，但标记会话异常。
  - 复盘保存失败时允许用户重新保存。

### 8.7 用户退出对话页

- 如果当前会话已有用户消息：
  - 弹窗提示“是否保留本次未完成练习？”
  - 操作：
    - `保留`
    - `丢弃`
    - `取消`
- 如果还没有用户消息：
  - 直接退出并丢弃会话。

## 9. 暂不实现的功能

### 9.1 第一版不做

- 用户注册/登录。
- 云端同步。
- 付费系统。
- 社区、评论、分享广场。
- 真人陪练。
- 语音对话。
- 视频虚拟人。
- 复杂心理测评。
- 社交能力等级体系。
- 排行榜、勋章、连续打卡。
- 用户自定义复杂剧本。
- 自动生成无限场景。
- 恋爱专项课程。
- 销售/商务谈判专项课程。
- 多人模拟。
- 通讯录或真实社交平台接入。

### 9.2 暂不做的原因

| 功能 | 暂不做原因 |
| --- | --- |
| 语音对话 | 语音识别、TTS、实时体验成本高，会拖慢 MVP |
| 用户系统 | 自用场景不需要，增加开发和安全成本 |
| 社区 | 没有用户来源，且内容治理成本高 |
| 课程体系 | 容易把产品做重，MVP 先验证练习闭环 |
| 无限场景生成 | 容易导致场景质量不稳定，先用固定高频场景 |
| 游戏化 | 不是核心价值，后续根据使用频率再判断 |

## 10. 验收标准

### 10.1 场景库验收

- App 启动后能看到 12 个固定场景。
- 场景可按分类筛选。
- 每个场景详情页都展示背景、角色、目标、难度、建议轮数。
- 点击 `开始练习` 能创建新会话。

### 10.2 对话练习验收

- 进入对话页后能展示 AI 开场白。
- 用户输入文本后能发送。
- 发送中输入框不可重复提交。
- AI 回复成功后追加到消息列表。
- API 失败时能重试。
- 对话消息能保存到本地。
- 用户能主动结束并进入复盘流程。

### 10.3 复盘验收

- 复盘页能基于完整对话生成结构化结果。
- 复盘至少包含：
  - 总体评价
  - 5 个维度评分
  - 至少 1 个暴露问题
  - 至少 1 个关键片段分析
  - 至少 2 条替代表达
  - 1 条下次练习建议
- 复盘结果能保存到历史记录。
- 用户能收藏替代表达。

### 10.4 历史记录验收

- 完成复盘后，历史记录页出现该条记录。
- 历史记录按时间倒序排列。
- 点击记录能查看完整复盘。
- 历史复盘中能查看完整对话。
- 用户能删除单条记录。

### 10.5 表达库验收

- 收藏表达后，表达库中能看到该表达。
- 表达展示内容、标签、来源场景、收藏时间。
- 用户能按标签筛选。
- 用户能复制表达。
- 用户能删除表达。

### 10.6 设置验收

- 用户能保存 API 地址、API Key、模型名称。
- API Key 本地存储不能明文展示。
- 用户能测试连接。
- API 未配置时，开始练习或生成回复会提示去设置。
- 用户能清空练习记录和表达库。

### 10.7 异常状态验收

- 无历史记录时展示空状态。
- 无收藏表达时展示空状态。
- 网络失败时不崩溃，并提供重试入口。
- 复盘生成失败时不丢失原对话。
- 用户中途退出对话时有确认提示。

### 10.8 MVP 完成定义

满足以下条件即可认为 MVP 完成：

1. 用户可以从 12 个场景中选择任意一个开始练习。
2. 用户可以完成一轮 6-10 轮文字对话。
3. App 可以输出可读、具体、结构化的复盘。
4. 用户可以回看历史复盘。
5. 用户可以收藏和复制表达。
6. 全流程不依赖登录、付费或云端用户系统。

## 11. V1.1/V1.2 扩展功能

MVP 已验证“场景选择 - 文字对话 - 复盘 - 表达沉淀”的核心闭环。后续扩展优先围绕两个方向：

1. 降低模型接入成本和配置门槛，支持多渠道模型。
2. 降低练习时的输入负担，逐步支持语音对话。

扩展功能仍保持自用、本地优先、不做账号系统、不做后端代理的边界。若未来公开发布，再重新评估 API Key 安全、隐私政策和后端代理。

### 11.1 多渠道模型配置

#### 目标

支持用户在 App 内保存多组模型服务配置，并分别选择“对话模型”和“复盘模型”。优先支持 OpenAI-compatible 接口，因此 DeepSeek、通义千问、Kimi、智谱、豆包等国产模型可以通过统一请求层接入。

#### 支持渠道

第一批提供以下预设：

| 渠道 | 类型 | 配置方式 | 说明 |
| --- | --- | --- | --- |
| DeepSeek | openai_compatible | 预设 API 地址，用户填写 Key 和模型名 | 优先推荐，成本低，中文能力较好 |
| 通义千问/Qwen | openai_compatible | 预设 API 地址，用户填写 Key 和模型名 | 适合中文对话和复盘 |
| Kimi | openai_compatible | 预设 API 地址，用户填写 Key 和模型名 | 长上下文复盘可作为备选 |
| 智谱 GLM | openai_compatible | 预设 API 地址，用户填写 Key 和模型名 | 国产模型备选 |
| 豆包 | openai_compatible | 预设 API 地址，用户填写 Key 和模型名 | 国产模型备选 |
| 自定义 | openai_compatible | 用户填写完整 API 地址、Key、模型名 | 兼容其他服务商或本地代理 |

具体 API 地址和模型名以服务商当前文档为准，App 内只提供可编辑预设，不把预设当作强约束。

#### 配置字段

每个渠道配置包含：

- `id`
- `provider_type`
- `display_name`
- `api_base_url`
- `api_key_alias`
- `chat_model_name`
- `review_model_name`
- `is_default_for_chat`
- `is_default_for_review`
- `enabled`
- `created_at`
- `updated_at`

其中 `api_key_alias` 只作为本地加密存储索引，真实 API Key 不写入 DataStore 或 Room 明文字段。

#### 页面交互

设置页从单一“API 配置”升级为“模型渠道”：

- 展示已配置渠道列表。
- 支持新增渠道。
- 支持从预设渠道创建配置。
- 支持编辑 API 地址、API Key、对话模型、复盘模型。
- 支持测试连接。
- 支持设为默认对话渠道。
- 支持设为默认复盘渠道。
- 支持禁用或删除渠道。
- 若没有可用渠道，开始练习时引导进入设置页。

#### 调用规则

- 对话练习使用默认对话渠道。
- 复盘生成使用默认复盘渠道。
- 如果默认复盘渠道未配置，则回退到默认对话渠道。
- 单次请求失败时提示当前渠道错误，不自动切换，避免不同模型导致上下文行为不可控。
- 后续可增加“临时切换本次练习模型”，但不作为 V1.1 必做。

#### 验收标准

- 用户至少可以保存 2 个不同渠道配置。
- 每个渠道的 API Key 独立加密保存。
- 可以分别设置默认对话模型和默认复盘模型。
- 测试连接能明确展示成功或失败原因。
- 删除当前默认渠道时，App 能提示重新选择默认渠道。
- 老版本单一 API 配置可以迁移为一个自定义渠道。

### 11.2 语音输入与回复播报

#### 目标

在不改变现有文字对话主流程的前提下，增加低成本语音能力：

1. 用户可以用语音输入内容。
2. AI 回复可以自动或手动播报。
3. 对话记录仍以文字保存。

#### 语音输入

- 对话输入区增加麦克风按钮。
- 点击后调用 Android 系统语音识别能力，将语音转为文本。
- 识别出的文本先填入输入框，用户可以编辑后发送。
- 可选设置：识别完成后自动发送。
- 识别失败时提示重新录入，不影响已保存消息。

#### 回复播报

- AI 回复气泡增加播放按钮。
- 对话页支持“自动播报 AI 回复”开关。
- 使用 Android TextToSpeech 进行本地播报。
- 支持停止当前播报。
- 后续可支持语速调节。

#### 数据与隐私

- V1.1 不保存原始音频。
- 消息表仍只保存最终文字内容。
- 复盘仍基于文字 transcript 生成。
- 语音识别是否走系统云服务由用户设备环境决定，设置页需要说明。

#### 验收标准

- 用户可以通过语音生成文本并发送。
- 识别结果可以手动修改。
- AI 回复可以点击播放。
- 自动播报开关可保存。
- 退出对话页时能停止播报。
- 未授权麦克风权限时有明确提示。

### 11.3 实时语音陪练模式

实时语音陪练作为 V1.2 或更后版本，不与 V1.1 一起开发。该模式的目标是让用户接近真实现场地开口练习，但实现复杂度明显高于语音输入。

#### 预期能力

- 独立的“语音练习模式”入口。
- 按住说话或免手动轮流说话。
- 识别中、思考中、播报中三种明确状态。
- 支持打断 AI 播报。
- 支持一轮结束后自动进入下一轮。
- 对话结束后仍生成文字复盘。

#### 暂不做内容

- 不做真人语音克隆。
- 不做视频角色。
- 不做多人语音房间。
- 不保存完整录音。
- 不做实时情绪识别。

### 11.4 其他可添加功能

以下功能优先级低于多渠道模型和语音输入，但与产品目标高度相关：

| 功能 | 价值 | 建议优先级 |
| --- | --- | --- |
| 本轮训练重点 | 用户开始前选择“开场、接话、拒绝、收尾”等重点，复盘更聚焦 | 高 |
| 表达卡片复习 | 收藏表达后可跟读、改写、迁移到新场景 | 高 |
| 自定义场景轻量版 | 用户填写“我想练什么场景”，由现有结构生成一次性练习 | 中 |
| 请求日志与成本估算 | 自用多模型时方便判断哪个渠道稳定、便宜 | 中 |
| 复盘导出 | 将一次练习导出为 Markdown，方便长期整理 | 中 |
| 难度调节 | 同一场景支持轻松、正常、压力三档 | 中 |
| 训练计划 | 每周推荐 3 个场景，减少选择成本 | 低 |

### 11.5 扩展功能完成定义

V1.1 完成定义：

1. 用户可以配置多个模型渠道。
2. 用户可以分别选择对话模型和复盘模型。
3. 用户可以用语音输入并编辑后发送。
4. 用户可以播放 AI 回复。
5. 文字对话、复盘、历史、表达库原有闭环不退化。

V1.2 完成定义：

1. 用户可以进入独立语音练习模式。
2. App 可以稳定处理听写、生成回复、播报、停止播报的状态切换。
3. 语音练习结束后可以生成与文字练习一致质量的复盘。
