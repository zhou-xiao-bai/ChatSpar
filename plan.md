# 社交练习生 App 开发计划

## 1. 计划目标

本计划基于 `prd.md`，目标是用最简单、稳定、低成本的方式完成一个安卓 MVP：

- 固定 12 个社交场景。
- 用户通过文字与 AI 进行场景对话。
- 对话结束后生成结构化复盘。
- 本地保存历史记录和收藏表达。
- 不做登录、不做后端、不做云同步、不做付费。

第一阶段只追求自用可安装、可连续练习、可复盘。

## 2. 技术路线

### 2.1 推荐技术栈

| 层级 | 技术 |
| --- | --- |
| 客户端 | Android 原生 |
| 语言 | Kotlin |
| UI | Jetpack Compose + Material 3 |
| 架构 | 单模块 MVVM + Repository |
| 状态管理 | ViewModel + StateFlow |
| 异步 | Kotlin Coroutines |
| 本地数据库 | Room |
| 设置存储 | DataStore |
| API Key 存储 | Android Keystore 加密后保存 |
| 网络请求 | Retrofit 或 Ktor Client + OkHttp |
| AI 接入 | OpenAI-compatible Chat API |
| 测试 | JUnit、Room in-memory test、ViewModel unit test |
| 发布 | 先生成 APK 自用安装 |

### 2.2 选型原因

- 原生 Android 对自用 APK、设备能力、本地数据库和系统存储支持最好。
- Compose 可以快速完成对话页、列表页、表单页和复盘页。
- Room + DataStore 足够覆盖 MVP 的本地数据需求。
- 不做后端可以减少部署、鉴权、账号和运维成本。
- AI API 通过接口抽象，未来可以平滑替换为自己的后端代理。

## 4. 阶段计划

### 阶段 0：准备

目标：明确设计参考、技术边界和任务拆分。

交付：

- `prd.md`
- `plan.md`
- `tasks.md`
- UI 参考图提示词和图片存放路径

完成标准：

- 每个页面都有对应 UI 参考图提示词。
- 每个开发任务都有验收标准和回归测试清单。

### 阶段 1：项目骨架

目标：跑通 Android 工程、Compose、导航、主题和基础页面壳。

交付：

- Android 项目可启动。
- 底部 3 个 Tab：练习、复盘、表达库。
- 设置入口。
- Debug APK 可安装。

完成标准：

- `./gradlew test` 通过。
- `./gradlew assembleDebug` 通过。
- 真机或模拟器能打开 App。

### 阶段 2：场景库

目标：完成 12 个固定场景的读取、展示和详情页。

交付：

- `scenarios.json`
- 场景列表页
- 分类筛选
- 场景详情页

完成标准：

- 12 个场景完整展示。
- 分类筛选正确。
- 点击场景可进入详情。

### 阶段 3：本地数据与设置

目标：完成本地数据基础设施和 AI API 配置。

交付：

- Room 数据库。
- 会话、消息、复盘、表达表。
- 设置页。
- API 地址、API Key、模型名保存。
- 测试连接能力。

完成标准：

- 设置可保存、读取、更新。
- API 未配置时能阻止开始 AI 请求。
- 数据库单元测试通过。

### 阶段 4：对话练习

目标：完成一轮文字对话。

交付：

- 对话练习页。
- 创建会话。
- 保存用户消息和 AI 消息。
- 发送中、失败、重试状态。
- 轮数提示。

完成标准：

- 用户可完成至少 6 轮对话。
- 中途退出有确认。
- 网络失败不丢失已有消息。

### 阶段 5：复盘闭环

目标：完成对话结束后的结构化复盘。

交付：

- 复盘生成。
- JSON 解析。
- 复盘结果页。
- 关键片段分析。
- 推荐表达收藏。

完成标准：

- 复盘至少包含总体评价、5 项评分、问题、关键片段、替代表达、下次建议。
- 复盘失败可重新生成。
- 完成复盘后历史记录可见。

### 阶段 6：历史与表达库

目标：完成历史回看和表达沉淀。

交付：

- 历史记录页。
- 历史复盘详情页。
- 完整对话展开。
- 表达库。
- 表达复制、筛选、删除。

完成标准：

- 完成练习后可回看。
- 收藏表达可在表达库查看和复制。
- 删除有确认。

### 阶段 7：稳定性与发布

目标：整理异常状态，生成可自用安装包。

交付：

- 空状态。
- API 失败状态。
- 复盘解析失败兜底。
- 数据清空能力。
- Release APK。

完成标准：

- 核心流程在无数据、有数据、网络失败、API 未配置场景下不崩溃。
- 可安装 APK 可完成完整练习闭环。

### 阶段 8：多渠道模型配置

目标：把单一 API 配置升级为多渠道模型配置，支持 DeepSeek、通义千问、Kimi、智谱、豆包和自定义 OpenAI-compatible 服务。

交付：

- 多渠道配置领域模型。
- 多 API Key 加密存储。
- 设置页模型渠道管理。
- 默认对话渠道和默认复盘渠道。
- AI 请求按用途路由到对应渠道。
- 老版本单一配置迁移。

完成标准：

- 至少可以保存 2 个渠道配置。
- 每个渠道可以独立测试连接。
- 对话和复盘可以使用不同模型。
- 删除或禁用默认渠道时有明确提示。
- `./gradlew test` 和 `./gradlew assembleDebug` 通过。

### 阶段 9：语音输入与播报

目标：在现有文字对话流程上增加轻量语音能力，先做语音输入和 AI 回复播报，不做实时语音通话。

交付：

- 麦克风权限处理。
- 对话页语音输入按钮。
- Android SpeechRecognizer 转文字。
- 识别结果回填输入框。
- AI 回复 TextToSpeech 播报。
- 自动播报开关。
- 退出页面时停止识别和播报。

完成标准：

- 用户可以通过语音生成文本并发送。
- 用户可以编辑识别文本后再发送。
- AI 回复可以手动播放。
- 自动播报开关可保存。
- 未授权权限、识别失败、TTS 不可用时不崩溃。

### 阶段 10：练习效果增强

目标：提升重复练习价值，让用户不仅完成单次对话，还能围绕表达和训练重点持续改进。

交付：

- 本轮训练重点选择。
- 难度档位调节。
- 表达卡片复习。
- 收藏表达跟读或改写练习。
- 复盘导出 Markdown。
- 请求日志与基础成本估算。

完成标准：

- 训练重点会进入对话 Prompt 和复盘 Prompt。
- 表达库中的表达可以被重新练习。
- 复盘导出内容包含场景、完整对话、评分、问题、建议表达。
- 请求日志不记录 API Key，不泄露敏感信息。

### 阶段 11：实时语音陪练模式

目标：在轻量语音能力稳定后，再提供独立语音练习模式，让用户接近真实现场地开口练习。

交付：

- 独立语音练习入口。
- 语音识别、AI 思考、TTS 播报的状态机。
- 支持停止播报。
- 支持连续轮次练习。
- 语音练习结束后复用现有复盘流程。

完成标准：

- 用户可以完成至少 6 轮语音练习。
- 状态切换清晰，不出现重复录音或重复播报。
- 退出页面时能释放录音和 TTS 资源。
- 语音练习生成的复盘质量不低于文字练习。

## 5. 任务开发规则

每个任务必须可以被单独开发、单独验收、单独回归。

### 5.1 单任务边界

每个任务必须写清楚：

- 输入：依赖哪些文档、模型或前置任务。
- 范围：这次只改什么。
- 不做：明确排除哪些事情。
- 交付物：新增或修改哪些页面、类、表、文件。
- 验收标准：什么叫完成。
- 回归测试：完成后必须执行哪些检查。

### 5.2 完成定义

一个任务完成必须满足：

1. 功能可运行。
2. 不破坏已有核心流程。
3. 有必要的单元测试或手动回归记录。
4. 没有无关重构。
5. 没有把后续任务提前混进来。

### 5.3 通用回归测试

除非任务说明另有豁免，每个任务完成后至少执行：

```bash
./gradlew test
./gradlew assembleDebug
```

涉及 UI 的任务，还需要手动检查：

- 页面可打开。
- 返回键行为正确。
- 空状态可见。
- 加载状态不遮挡主要操作。
- 按钮不会重复提交。
- 文案不超出容器。

涉及数据库的任务，还需要检查：

- 新增数据能查询。
- 更新数据能保存。
- 删除数据不残留异常引用。
- App 重启后数据仍存在。

涉及 AI 的任务，还需要检查：

- API 未配置时不发请求。
- 请求失败可重试。
- 返回空内容有提示。
- 解析失败不导致崩溃。

## 7. 第一阶段范围

第一阶段只做：

- Android 项目骨架。
- 12 个固定场景。
- 文字对话。
- 本地 API 配置。
- AI 回复。
- AI 复盘。
- 历史记录。
- 表达收藏。
- 本地数据保存。
- Debug/Release APK。

第一阶段不做：

- 登录注册。
- 后端服务。
- 云同步。
- 语音对话。
- 视频角色。
- 社区。
- 付费。
- 无限场景生成。
- 课程系统。

## 8. 主要风险与处理策略

| 风险 | 影响 | 处理策略 |
| --- | --- | --- |
| AI 复盘空泛 | 用户觉得没帮助 | Prompt 必须要求引用用户原话，输出关键片段分析 |
| AI JSON 解析失败 | 复盘页无法展示 | 允许重试，保存原始返回，增加兜底解析 |
| 客户端保存 API Key | 公开发布有安全风险 | 自用阶段接受；公开发布前改为后端代理 |
| 对话上下文太长 | 成本和延迟增加 | 限制 6-10 轮，MVP 不支持超长对话 |
| UI 做复杂 | 拖慢上线 | 只做工具型页面，不做营销视觉 |
| 场景质量不足 | 练习不真实 | 固定 12 个场景精写，不做无限生成 |
| 多渠道配置混乱 | 用户不知道当前请求使用哪个模型 | 设置中明确区分默认对话渠道和默认复盘渠道，请求失败时展示渠道名 |
| 国产模型兼容差异 | 不同服务商 OpenAI-compatible 细节不同 | 先支持 chat completions 基础字段，预设可编辑，失败时保留原始错误摘要 |
| 多 API Key 管理风险 | Key 混用或删除后残留 | 使用 alias 隔离加密存储，删除渠道时同步清理对应 Key |
| 语音识别不稳定 | 影响练习连续性 | 第一阶段只把识别结果回填输入框，用户确认后发送 |
| TTS 生命周期泄露 | 退出页面仍在播报 | ViewModel 或页面生命周期统一停止和释放 TTS |
| 实时语音复杂度过高 | 状态失控、调试困难 | 先完成语音输入和播报，再单独实现实时语音状态机 |
